- `.codex/skills/tcal-dimer-extractor/SKILL.md`: tcal dimer extractor のプロダクト・実装・検証方針。
- `.codex/skills/tcal-agent-delegation/SKILL.md`: サブエージェント委任と、委任プロンプトを誤って最終回答に出さないための手順。

サブエージェントへ委任する場合は、実際に利用可能な委任手段だけを使う。委任用 JSON や内部プロンプトをユーザー向け最終回答として出力してはいけない。利用可能な委任手段がない場合は、その制約を明示し、作業範囲を小さくして親エージェントが安全に進める。

# Agent Guidelines

このリポジトリは、tcal 用のダイマー構造を CIF から視覚的に抽出する GitHub Pages 対応 Web GUI を作るためのものです。

## Project Purpose

結晶構造 CIF ファイルをブラウザで読み込み、ユーザーがセル拡張と分子選択を視覚的に行い、tcal で使えるダイマー `xyz` ファイルを再現可能に出力する。

目標ユーザーは、有機半導体などの結晶構造から tcal 計算用の分子ダイマーを手早く作りたい研究者である。Mercury や GaussView を手作業で往復する部分を、ブラウザ内の選択・命名・エクスポートに置き換える。

## tcal Alignment

- tcal は transfer integral 計算用プログラムであり、PySCF モードではダイマー構造の `xyz` を入力にする。
- 出力する `xyz` は、原則として前半を中心分子、後半を選択した周辺分子の原子順にする。
- 異種分子や原子数が異なるダイマーを扱う場合に備え、中心分子の原子数を再現情報に保持する。
- このアプリは tcal 計算そのものを実行しない。責務は CIF から tcal 入力候補の `xyz` と再現レシピを作るところまで。

## Product Requirements

- 静的サイトとして GitHub Pages で動作すること。通常の利用にサーバーを要求しない。
- CIF ファイルはブラウザ内で処理し、ユーザーの構造ファイルを外部サーバーへ送らない。
- ユーザーは GUI でセル拡張範囲を指定できる。
- ユーザーは可視化された分子集合から中心分子を 1 つ選択できる。
- ユーザーは中心分子に対する周辺分子を複数選択できる。
- 選択した各周辺分子には、ユーザーが `A`, `B`, `pi-stack`, `herringbone` などの任意名を付けられる。
- 各選択は、中心分子 + 周辺分子の 1 ダイマーとして `xyz` 出力できる。
- 複数ダイマーをまとめて ZIP などでエクスポートできる設計を優先する。
- 再現用のメタデータをエクスポートし、同じ CIF とメタデータからセル拡張・中心分子・周辺分子・命名・出力を復元できるようにする。

## Reproducibility Contract

再現情報は、人間が読めて機械でも読める形式にする。実装では JSON を第一候補にする。

保持すべき情報:

- 元 CIF ファイル名と、可能なら内容ハッシュ
- セル拡張範囲
- 分子分割に使った規則
- 中心分子の識別子
- 周辺分子の識別子、ユーザー定義名、中心分子との関係情報
- `xyz` の原子順序
- 中心分子の原子数
- アプリのバージョンまたは recipe schema version

CIF に埋め込む場合は、元の結晶構造データを壊さないコメントブロックまたは専用 data item を使う。CIF への埋め込みが複雑になる場合は、無理に一体化せず、元 CIF と recipe JSON のペアを正とする。

## UI Principles

- 最初の画面から CIF 読み込み、セル拡張、分子選択、命名、出力まで進める実用画面にする。マーケティング用ランディングページは作らない。
- 研究用途の操作画面として、情報密度は高めでよいが、選択状態と出力対象は常に明確にする。
- 中心分子、選択済み周辺分子、未選択分子は視覚的に区別する。
- 分子選択の取り消し、リネーム、出力プレビューは必須に近い基本操作として扱う。
- 3D 表示は空白や重なりがないことを実機ブラウザで確認する。

## Engineering Principles

- CIF 解析、周期境界、分子分割、3D 表示は、実績のある既存ライブラリを優先する。
- 文字列の ad hoc parse で CIF を処理しない。妥当な構造化パーサーがある場合はそれを使う。
- 浮動小数点座標の丸めは、出力直前に一箇所で行う。
- ダイマー抽出の内部表現と UI 状態を混ぜない。再現 recipe は UI の一時状態ではなく、明示的なドメインデータから作る。
- ファイル名は安全な ASCII を基本にし、ユーザー入力名は必要に応じて slug 化する。

## Testing And Verification

- CIF 読み込み、セル拡張、分子分割、ダイマー `xyz` 出力、recipe 復元はユニットテスト対象にする。
- 代表 CIF を小さな fixture として持ち、出力原子数・原子順・中心分子原子数を検証する。
- GUI 変更後は、デスクトップとモバイル幅でブラウザ確認を行う。
- 検証や実機ブラウザ確認には `$Browser-Use` を使う。
- 3D 表示を実装した場合は、canvas が非空で、中心分子と選択分子が判別できることを確認する。
- GitHub Pages 配信用の build が通ることを完了条件に含める。
- push 前には import 整理とフォーマットのために `pnpm biome check . --writes` を実行する。このコマンドで lint エラーが出ても、lint 合否確認としては扱わず無視してよい。
